# SyncWalk

SyncWalk is a full-stack audio tour platform with solo and group listening modes.

## Tech stack

- Frontend: Next.js, TypeScript, Tailwind CSS, Zustand, Socket.IO client
- Backend: FastAPI, SQLAlchemy, Alembic, Redis, PostgreSQL, Socket.IO server
- Infrastructure: Docker Compose, Nginx
- Payments: WayForPay

## Project structure

- `frontend` - web app for authentication, tours, payments, and room sessions
- `backend` - API, business logic, websocket handling, and database migrations
- `nginx` - reverse proxy and SSL-related configuration
- `scripts` - deployment, backup, restore, and SSL helper scripts
- `docs` - project documentation and operational notes

## Prerequisites

- Docker and Docker Compose
- Node.js 20+ with `pnpm` (for local frontend development without Docker)
- Python 3.11+ (for local backend development without Docker)

## Environment setup

1. Copy `.env.example` to `.env`.
2. Fill in required values for database, Redis, JWT, and WayForPay credentials.
3. Verify `NEXT_PUBLIC_API_URL` and backend domain-related variables match your environment.

## Run with Docker

```bash
docker compose up --build
```

Default services:

- Frontend: `http://localhost:3000`
- Backend API: `http://localhost:8000`

## Local development

Backend:

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

Frontend:

```bash
cd frontend
pnpm install
pnpm dev
```

## Database migrations

```bash
cd backend
alembic upgrade head
```

## Makefile usage

The project includes a `Makefile` to simplify common Docker and operational commands.

Basic usage:

```bash
make <target>
```

Common development targets:

- `make dev` - restart and rebuild the full development stack
- `make dev-up` - start the development stack with build
- `make dev-down` - stop the development stack
- `make down` - alias for `make dev-down`
- `make logs` - follow API and frontend logs
- `make logs-api` - follow API logs only
- `make logs-frontend` - follow frontend logs only

Database and backend targets:

- `make migrate` - apply latest Alembic migrations in dev stack
- `make migration msg="your message"` - create a new autogenerated migration
- `make seed` - seed database with initial data
- `make shell-api` - open a shell inside the API container
- `make shell-db` - open `psql` inside the Postgres container

Production-oriented targets:

- `make prod` - restart and rebuild the production stack in detached mode
- `make prod-up` - start production stack in detached mode with build
- `make prod-down` - stop production stack
- `make prod-migrate` - run migrations in production stack
- `make prod-logs` - follow API, frontend, and nginx logs

Ops helpers:

- `make backup` - run database backup script
- `make deploy` - run deployment script
